Version 2.0.4 

  • Application

    • You can now preview Mod Pack files while game is running
    • You can now select default race to load from customize menu
    • Added Keep Camera Position option to prevent model from zooming out when loaded
    • Now checks for incorrect index dat counts on startup
      • This prevents the "Could not locate data files." error
    • Now checks for outdated index backups in problem check and when doing a Start Over
      • This seems to be the cause of certain models missing in-game and several game crashes
    • Updated to latest Helix Toolkit release
      • Hopefully this solves the black screen issue
    • Additional messages to handle more unhandled exceptions
    • Now creates crash dump files to %LOCALAPPDATA%\FFXIV_TexTools\CrashDumps

  • 3D

    • Index import calculations now match 1.9.9.3
    • Improved handling of bones on import
    • Mesh parts are now automatically reorganized (eg. [0, 0.1, 0.4] > [0, 0.1, 0.2])

  • Bug Fixes

    • Fix for issue where UV seams would break upon import, causing texture to appear distorted
    • Fix for issue where error would appear if Mesh 0 was blank or missing
    • Fix for issue where TT would not get correct data if meshes in the DAE file were not in order
    • Fix for issue where bones that were not used were being imported, causing distortions or meshes to not appear in-game
    • Fix for issue where unorganized mesh parts in the DAE would cause "given key not present" error on import
    • Fix for issue where DAE files with large amounts of vertex color data would make the application hang
    • Fix for issue where DAE read would fail if meshes had different types for indices
    • Fix for issue where mdl header could not be larger than 384bytes causing large models to fail
    • Fix for issue were DAE could have extra values added on to vertex colors, which would cause parts of a mesh to appear black in-game
    • Other small fixes
